# CLI Reference

Every command writes its artifact under `output/<category>/` and prints the path. Shared
good-fraction options appear on most commands: `--max-size` (default 24), `--max-prime` (default
5), `--max-lcm` (default 24), `--ktet` (default 12).

## Tables

### `good-fractions`

```bash
melodroid table good-fractions [--max-size 24] [--max-prime 5]
```

Prints the good fractions for the given size/prime limits.

### `lcm-families`

```bash
melodroid table lcm-families [--max-size 24] [--max-prime 5] [--max-lcm 24]
```

Prints the LCM families, ascending by LCM, empty families omitted.

### `bin-overlaps`

```bash
melodroid table bin-overlaps [--max-size 24] [--max-prime 5]
```

For each adjacent pair of good fractions, the bin radius
$c = (\text{Upper} - \text{Lower}) / (\text{Upper} + \text{Lower})$ at which their JND bins first
touch. The caption reports the minimum $c$ — the binding constraint.

### `octave-sweep`

```bash
melodroid table octave-sweep --ratios 1.0 1.25 1.5 \
  [--sweep-step 0.001] [--bin-radius 1/161] [--only-full-matches] \
  [--only-centered-full-matches] [--max-size 24] [--max-prime 5]
```

Sweeps a reference ratio across $[1, 2)$, binning each input ratio against the good fractions;
flags full matches and centered full matches. `--ratios` is required (space-separated decimals on
$[1, 2)$).

Strict full matches are highlighted green. When a ratio falls in two overlapping bins
(only possible at radius $\geq 1/161$ for defaults), the row is flagged ambiguous: each competing column is
filled with a `~` prefix and the row is highlighted yellow; its LCM cell shows the **minimum**
LCM across candidate readings with a `?` suffix (e.g. `9?`).

Some common reference sets:

- (LCM 4) `1.0 1.25 1.5`
- (LCM 8) `1.0 1.125 1.25 1.5 1.875`

### `ktet-cutoffs`

```bash
melodroid table ktet-cutoffs [--max-k 50] [--only-strictly-improving] \
  [--max-size 24] [--max-prime 5]
```

Lists the worst-case covering radius $c_k$ per $k$, its limiting fraction, and the nearest key.

### `key-sweep`

```bash
melodroid table key-sweep --ktet 12 (--keys 0 4 7 | --ratios 1.0 1.25 1.5) \
  [--bin-radius c_k] [--only-full-matches] [--max-size 24] [--max-prime 5]
```

Like the octave sweep but stepping through the $k$ keys of the tuning. At least one of `--keys` /
`--ratios` is required; both may be combined.

### `placement`

```bash
melodroid table placement --lcm 4 [4 ...] --at 0 \
  [--ktet 12] [--max-size 24] [--max-prime 5] [--max-lcm 24]
```

Prints the keys and fractions for one or more LCM placements at a shared anchor.

### `family-overlap`

```bash
melodroid table family-overlap --lcm-sweep 4 --lcm-ref 24 \
  [--ktet 12] [--max-size 24] [--max-prime 5] [--max-lcm 24]
```

Sweeps one LCM shape against a reference family; green rows sit wholly inside the reference.

### `key-supersets`

```bash
melodroid table key-supersets --keys 0 4 7 \
  [--ktet 12] [--max-size 24] [--max-prime 5] [--max-lcm 24]
```

Lists every placement whose keys contain the requested set, tightest fits first. `--keys` accepts any integer and octave-normalizes it into `[0, ktet-1]` (e.g. `14 → 2`, `-1 → ktet-1`).

### `superpositions`

```bash
melodroid table superpositions --keys 0 2 4 5 7 9 11 \
  [--min-block-lcm 2] [--max-block-lcm 24] [--max-results 200] [--any-reference] \
  [--ktet 12] [--max-size 24] [--max-prime 5] [--max-lcm 24]
```

Lists minimal/irredundant ways to cover `--keys` as a union (superposition) of LCM-family placements, fewest pieces first. A building block may extend past the target; each row reports its extra keys (union minus target) and whether the pieces partition the target (`Disjoint?`). Every piece uniquely covers at least one target key. `--keys` octave-normalizes like `key-supersets`. Restrict the building blocks with `--min-block-lcm` / `--max-block-lcm` — e.g. `--min-block-lcm 8 --max-block-lcm 12` composes the target from the 8/9/10/12 families alone. `--max-results` caps the listing.

By default only **unique-reference** decompositions are listed: every piece shares one reference key (anchor), shown in the `Ref` column. The reference of a placement `L@a` is its anchor `a` — the key where the family's fundamental (`1/1`) sits — so a unique-reference cover roots the whole sound on a single fundamental. Isomorphic families are handled as aliases: the same keyboard key-set reachable by different families at shifted anchors (e.g. `4@0` and `3@7`, or `8@0` and `12@7`) lets a piece join whichever reference its partners share. With `--min-block-lcm 8 --max-block-lcm 12` this exposes the major scale as `8@0 + 12@0` (rooted on key 0, no extras) — the shared-reference alias of the cross-anchor `8@0 + 8@5`. Because the pieces share a fundamental, the whole superposition's wave pattern length is the `lcm` of their family LCMs, shown in the `LCM` column: for `8@0 + 12@0` that is `lcm(8, 12) = 24` — the very pattern of the single-piece `24@0` cover.

Pass `--any-reference` to drop the shared-reference requirement and list every minimal cover (pieces may sit at different anchors, the `Ref` column is omitted); that is where mixed-reference identities such as the major scale `= 8@0 + 8@5` and the natural minor `= 8@1 + 8@6` appear.

### `voicings`

```bash
melodroid table voicings (--lcm 24 | --keys 0 4 7) \
  [--ktet 12] [--max-size 24] [--max-prime 5] [--max-lcm 24]
```

Lowest-penalty voicings from each root. Exactly one of `--lcm` / `--keys`.

### `subsets`

```bash
melodroid table subsets (--lcm 15 [--at 0] | --keys 0 4 7) \
  [--ktet 12] [--bin-radius <c>] [--only-full-matches] [--max-results 200] \
  [--ambiguous-matches 1] [--max-size 24] [--max-prime 5] [--max-lcm 24]
```

Enumerates every size-≥2 subset of the base key set (an `--lcm@--at` placement or a literal
`--keys` set) and key-sweeps each one. Output is a matrix: one row per subset, one column per
reference key `n`, with each cell holding the LCM the subset full-matches at that reference. This
surfaces the renormalized-subset relations of the `graph lcm-families` output at the keyboard
level — e.g. `--lcm 15` reveals subsets that full-match LCM 20. Ambiguous full matches (marked
with a trailing `?`) are included by default; `--only-full-matches` restricts to strict matches.
An ambiguous cell can full-match more than one LCM family; `--ambiguous-matches` (default 1) sets
how many candidate LCMs each ambiguous cell lists, ascending best-fit (shortest wave pattern
length) first — e.g. `--ambiguous-matches 2` shows `8? 9?`. `--max-results` caps the number of subset rows. Exactly one of
`--lcm` / `--keys`. `--bin-radius` defaults to the `c_k` cutoff (as in `key-sweep`).

### `chord-melody`

```bash
melodroid table chord-melody --chord-keys 0 4 7 \
  [--ktet 12] [--max-size 24] [--max-prime 5] [--max-lcm 24]
```

Matrix of maximal-LCM placements containing the chord vs each key.

### `chords`

```bash
melodroid table chords \
  [--ktet 12] [--min-notes 2] [--max-notes 7] [--max-results 1000]
```

Enumerates the unique chords of a `--ktet` keyboard, deduplicated under **transposition**:
a chord and all its transpositions count once, so `0 4 7`, `0 3 8` and `0 5 9` are one chord.
Equivalently these are the binary *necklaces* of length `k` — for 12-tet there are 352 in
all (including the empty set and singletons), or 280 across the default 2–7 notes. Each row
lists the canonical key set (the most-compact rotation, transposed to start at 0 — the "normal
order" of set theory, e.g. the major triad as `0 4 7`), its interval gaps around the octave
(summing to `k`), and its orbit size (the number of distinct transpositions; symmetric chords
like the augmented triad `0 4 8` have small orbits). `--min-notes` / `--max-notes` filter by
note count; `--max-results` caps the rows. This command is self-contained — it does not touch
good fractions or LCM families.

## Graphs

### `lcm-families`

```bash
melodroid graph lcm-families [--mode full|collapsed] \
  [--max-size 24] [--max-prime 5] [--max-lcm 24]
```

Writes a Mermaid relation graph to `output/graphs/`. `--mode collapsed` groups each isomorphism
class into a single node.

## Plots

### `lcm-families`

```bash
melodroid plot lcm-families --lcm 6 [--mode all|sum|constituents|difference] \
  [--subset-lcm K] [--difference-only] [--samples-per-period 200] \
  [--max-size 24] [--max-prime 5]
```

Writes a ScottPlot PNG of the family's sines and their superposition to `output/plots/`. See [Wave Pattern Plots](./wave-pattern-plots.mdx) for worked `plot lcm-families` examples with interactive plots.
